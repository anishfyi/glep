### Task 1: search.rs fast path (compile-once, -l short-circuit, chunked streaming)

**Files:** Modify: `src/search.rs`

**Interfaces:** `search::run` signature unchanged. `search_one` becomes `search_one(matcher: &grep_regex::RegexMatcher, root: &Path, rel: &Path, opts: &SearchOpts) -> anyhow::Result<(Vec<u8>, bool)>`.

- [ ] **Step 1: compile the matcher once.** In `run()`, the existing up-front `build_matcher(pattern, opts)?` result becomes THE matcher: bind it, pass `&matcher` into the rayon closure (grep-regex's RegexMatcher is Sync). Delete the per-file `build_matcher` call inside `search_one`; it now takes the matcher as its first parameter. The `Err(_)` arm of the map closure stays (I/O races still map to no-match).

- [ ] **Step 2: -l short-circuits.** In `search_one`, when `opts.files_with_matches` is true, skip the printer entirely and run this sink, returning `(Vec::new(), sink.0)`:

```rust
struct FoundSink(bool);

impl grep_searcher::Sink for FoundSink {
    type Error = std::io::Error;
    fn matched(
        &mut self,
        _: &grep_searcher::Searcher,
        _: &grep_searcher::SinkMatch<'_>,
    ) -> Result<bool, std::io::Error> {
        self.0 = true;
        Ok(false) // stop at first match
    }
}
```

- [ ] **Step 3: chunked streaming output.** Replace collect-all-then-sort with bounded-memory chunks that preserve global order: iterate `files.chunks(128)`; for each chunk run the existing `par_iter().enumerate()` map, sort the chunk's results by index, print them (including the `--` separator logic across chunk boundaries: `printed_any` and the separator state must live OUTSIDE the chunk loop so a context separator still lands between the last match of chunk N and the first of chunk N+1). Return `found` accumulated across chunks.

- [ ] **Step 4: verify byte-identical behavior.** Run `cargo test --all`. Expected: 57 tests green, parity included, zero modifications to any test file. Also run a manual tripwire: in a throwaway temp dir, `glep -C 1 <pattern spanning >128 files? not needed>`; the suite plus parity covers it.

- [ ] **Step 5: Commit.**

```bash
git add src/search.rs
git commit -m "perf: compile matcher once, -l short-circuit, chunked streaming output"
```

### Task 2: index update fast path (skip no-op manifest write, HashMap diff)

**Files:** Modify: `src/index/mod.rs`

- [ ] **Step 1: HashMap for changed_old_ids.** In `update()`, before the fresh loop, build `let id_by_path: std::collections::HashMap<&Path, u32> = self.manifest.live_entries().map(|e| (e.path.as_path(), e.id)).collect();` and compute `changed_old_ids` via `fresh.iter().filter_map(|m| id_by_path.get(m.path.as_path()).copied()).collect();` replacing the O(fresh x live) `.find()` loop. (Borrow note: id_by_path borrows manifest immutably; ensure its last use precedes the tombstone mutation, exactly like by_path today.)

- [ ] **Step 2: skip the manifest rewrite when nothing changed.** In the `fresh.is_empty() && dead_ids.is_empty()` early path, update `last_sweep_epoch` in memory always, but call `self.manifest.save(...)` ONLY when `ttl_secs > 0` (the persisted epoch is consumed solely by ttl checks; skipping the write when ttl is unused makes the no-op query write-free). Add this test to the tests module:

```rust
    #[test]
    fn noop_update_without_ttl_writes_nothing() {
        let dir = corpus();
        let mut idx = Index::build(dir.path(), 1_048_576).unwrap();
        let mpath = dir.path().join(".glep/manifest.bin");
        let before = std::fs::metadata(&mpath).unwrap().modified().unwrap();
        idx.update(1_048_576, 0).unwrap();
        let after = std::fs::metadata(&mpath).unwrap().modified().unwrap();
        assert_eq!(before, after, "no-op update with ttl 0 must not rewrite the manifest");
    }
```

## Self-review notes

- Chunk size 128 balances parallelism (rayon saturates within a chunk) against memory (at most 128 buffered match outputs); candidates rarely exceed one chunk on narrow queries, so the common path is unchanged.
- The -l sink change removes printer work but keeps searcher semantics (binary detection, line counting) identical; existing `-l` tests and the parity `-l` case pin output.
- Skipping the manifest write changes only WHEN a file is written, never what queries return; the new test pins it.
